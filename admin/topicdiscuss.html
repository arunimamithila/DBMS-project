Real-world objects such as users and computers are represented as objects in Active Directory. And these objects can be contained in other objects called container objects. Active Directory Domain Services supports numerous object types, and each is represented by a unique global identifier. Each object has a set of attributes that best describe it. For example, consider a user object. Each user can have attributes such as Name, Address, and Telephone number. The objects that can be authenticated and to which permissions can be assigned are called security principals. Each security principal object has a security identifier (SID) associated with it, in addition to the global identifier (GUID).
